04 Pillar 1 — AI / Data Activation

The big one. David explicitly said he'll gauge how you position Boomi's value prop in the context of AI.

The 30-second pitch

Say: "Every Commercial customer right now has a CEO asking about AI. The problem isn't model access — that's solved. The problem is that their data lives in 12 systems, none of it is clean, and there's no governance layer for the agents they want to deploy. That's the gap Boomi closes. Integration moves the data, DataHub trusts it, APIM exposes it, and AgentStudio governs the agents that consume it. We're not selling AI — we're selling the runway that makes AI actually work in production."

Why "data activation" matters as a phrase

  • It's Steve Lucas's deliberate reframe — away from "integration" (commodity) toward "making data usable for AI" (board-level priority).
  • For a Commercial buyer, it translates as: "we get you to AI ROI without rebuilding your stack."
  • For an AE, it translates as: "bigger deals, faster TCV growth, AgentStudio attach on every renewal."

How to talk about AI governance

Don't be hand-wavy. Three concrete governance moments to name:

  • Identity & access: which agent can call which API, with which credentials, on whose behalf.
  • Data scoping: agents only see the data they're authorised to see (MCP servers enforce this).
  • Observability: every agent call is logged, traceable, replayable — Commercial CTOs need this for audit and renewal.
If David pushes: "Most iPaaS vendors are saying the same AI story. What's different?" — your answer is "the control plane is built, not promised". AgentStudio is GA, MuleSoft Topic Center is still positioning. That's a 12-month head start in Commercial deals.

05 Pillar 2 — Single Control Plane (AgentStudio + MCPs)

David's headline framing. Be able to explain this in one breath, then unpack it.

The one-breath version

Say: "AgentStudio is the single place a Commercial customer designs, deploys, observes and governs every AI agent they're running — regardless of which model or which MCP server those agents call. Same place you already build integrations. Same governance. One pane of glass."

What MCP is, in plain English

Model Context Protocol (MCP) — an open standard (originated by Anthropic) that lets AI agents discover and call tools and data sources in a uniform way. Think of it as USB-C for agents: instead of bespoke integration per agent per data source, an agent can talk to any MCP-compliant server.

Why Boomi cares: Boomi can expose every existing integration, every APIM endpoint, every DataHub entity as an MCP server — instantly making 20,000+ customers' data and APIs available to any agent, governed.

Why "single control plane" is the win

Without it (status quo)With AgentStudio
Agents built in 4 different tools (LangChain, custom, vendor-specific, internal).One place to design, version, deploy.
Governance handled (or not) per agent.One governance layer, enforced at runtime.
Observability fragmented across model providers, log systems, integration logs.Unified agent telemetry — what was called, by whom, with what result.
Each agent re-integrates with the same APIs and data sources.One MCP layer, every agent reuses it.
The Commercial sales motion: AgentStudio is the land-and-expand multiplier. Customer buys Boomi for integration, builds 5 processes. Six months later, AgentStudio attaches on top to deploy their first agent against those processes. Bigger TCV, stickier renewal.
